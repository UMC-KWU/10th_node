- 데이터베이스 정규화
    
    테이블 간에 중복된 데이터를 허용하지 않고 데이터를 올바른 테이블에 저장하는 것! 중복된 데이터를 허용하지 않음으로써 무결성을 유지할 수 있으며 DB 저장 용량을 줄일 수 있음
    
- 인덱스(Index)
    
    DB 테이블에서 특정 컬럼을 빠르게 찾기 위한 별도의 자료구조. 조회가 많은 컬럼에 걸면 좋고 무분별하게 걸면 오히려 INSERT/UPDATE/DELETE 할 때마다 수정돼야해서 오히려 느려짐
    
- ORM VS Raw SQL
    
    ORM : SQL을 직접 안 쓰고 코드로 DB를 다루는 것! 그냥 자바 코드로만 DB를 다룸 다만 복잡한 쿼리는 한계가 있음
    - 장점: 코드 간결, DB 독립적
    - 단점: 복잡한 쿼리 한계, 성능 예측 어려움
    
    Raw SQL: 배운 것과 같이 SQL을 직접 쓰는 것
    - 장점: 복잡한 쿼리 자유롭게 가능, 성능 최적화 가능
    - 단점: 코드가 길어짐, DB 바뀌면 SQL도 수정
