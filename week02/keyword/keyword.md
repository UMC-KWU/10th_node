# 🎯 핵심 키워드

- 데이터베이스 정규화
    
    : 데이터 중복을 최소화하고, 이상 현상(삽입/삭제/갱신 이상)을 방지하기 위해 테이블을 구조적으로 분해하는 과정이다.
    
    - 특징
        - 데이터를 여러 테이블로 나눈다.
        - 중복 데이터를 제거한다.
        - 데이터 무결성을 유지한다.
        - 외래키로 연결된다.
    - 장점
        - 데이터 중복 감소로 인해 저장 공간이 절약된다.
        - 데이터 수정 시 일관성이 유지된다.
        - 이상현상이 방지된다.
    - 단점
        - 테이블이 많아지므로 구조가 복잡해진다
        - join연산이 증가되어 조회 성능이 저하될 수 있다.
        - 설계 난이도가 상승한다
          
- 인덱스(Index)
    
    : 데이터베이스에서 특정 컬럼의 검색 속도를 빠르게 하기 위해서 사용되는 자료구조이다.
    
    - 특징
        - 인덱스를 통해 WHERE, JOIN, ORDER BY에서 성능이 향상됨
    - 장점:
        - 검색 속도가 빨라지고 대량데이터를 조회할때의 성능이 좋아짐
        - 정렬, 범위 검색에 효과적임
    - 단점:
        - INSERT, UPDATE, DELETE시 성능이 저하됨
        - 추가적인 저장 공간이 필요하고, 너무 많으면 오히려 성능 안좋아짐
          
- ORM VS Raw SQL
    
    → ORM은 코드와 데이터베이스를 자동으로 매핑해주는 기술
    
    따라서 SQL을 직접 작성하지 않아도 되며 객체 중심의 빠른 개발이 가능함. 
    
    →Raw SQL은 SQL을 직접 작성하여 데이터베이스를 제어하는 방식
    
    코드를 직접 작성하기 때문에 DB동작을 정확히 제어 가능함. 복잡한 쿼리를 작성하는데 용이함.

    | ---  | --- | ---     |
    |난이도 | 쉬움| 어려움   |
    | 성능  | 보통| 높음     |
    | 유지보수 | 쉬움 | 어려움 |
